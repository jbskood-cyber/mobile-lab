# FOCO Premium Interaction Research

The functional-core block applies current platform guidance rather than adding decorative complexity.

## Principles applied

- Feedback stays close to the action and communicates status, success, warnings and reversible mistakes.
- Haptics use short system patterns with consistent meanings and always accompany visible feedback.
- Controls use Android-friendly touch targets of at least 48 logical pixels where interaction density permits.
- Motion remains restrained, and custom sheets disable animation when the system requests reduced motion.
- Local data uses Expo SQLite key-value storage so changes persist across app restarts while remaining compatible with Expo Go.
- Empty and storage-error states preserve context instead of leaving blank screens.

## Official references

- Apple Human Interface Guidelines: Feedback, Playing haptics, Motion and Accessibility.
- Android Developers: Core app quality accessibility guidance.
- Expo SDK 54 documentation: Haptics and SQLite key-value storage.
